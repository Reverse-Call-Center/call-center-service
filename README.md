# Call Center Service
(This was generated by a bobot, sorry.)
A .NET 8 Web API backend service for managing virtual call center operations, including sessions, interactions, receipts, and brand management.

https://reversecallcenter.com/discord


## 🚀 Features

- **Session Management**: Handle call center sessions with unique identifiers
- **Interaction Tracking**: Track customer interactions and call details
- **Receipt Management**: Process and store transaction receipts

## 🛠 Technology Stack

- **.NET 8.0**: Latest LTS version of .NET
- **ASP.NET Core**: Web API framework
- **Entity Framework Core**: ORM with PostgreSQL provider
- **PostgreSQL**: Primary database
- **Swagger/OpenAPI**: API documentation
- **Docker**: Containerization support

## 📋 Prerequisites

- [.NET 8.0 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
- [PostgreSQL 15+](https://www.postgresql.org/download/)
- [Docker](https://www.docker.com/get-started) (optional, for containerized deployment)

## 🏃‍♂️ Getting Started

### Local Development

1. **Clone the repository**
   ```bash
   git clone https://github.com/Reverse-Call-Center/call-center-service.git
   cd call-center-service
   ```

2. **Set up the database connection**
   
   Update `src/appsettings.Development.json` or set the environment variable:
   ```json
   {
     "ConnectionStrings": {
       "DefaultConnection": "Host=localhost;Database=callcenter;Username=your_user;Password=your_password"
     }
   }
   ```

3. **Run database migrations**
   ```bash
   cd src
   dotnet ef database update
   ```

4. **Start the application**
   ```bash
   dotnet run
   ```

5. **Access the API**
   - API: `https://localhost:5000`
   - Swagger UI: `https://localhost:5000/swagger`

### Docker Development

1. **Build and run with Docker**
   ```bash
   cd src
   docker build -t call-center-service .
   docker run -p 5000:5000 call-center-service
   ```

## 📚 API Endpoints

The service provides REST APIs for managing:

### Sessions
- Manage call center sessions
- Track session state and metadata

### Interactions  
- Record customer interactions
- Store interaction details and outcomes

### Receipts
- Process transaction receipts
- Handle receipt validation and storage

### Brands
- Manage multi-brand operations
- Brand-specific configurations

> **Note**: Visit `/swagger` endpoint when running the application to see the complete API documentation with request/response schemas.

## 🗄️ Database Schema

The application uses Entity Framework Core with PostgreSQL and includes:

- **Sessions**: Call center session data
- **Interactions**: Customer interaction records  
- **Receipts**: Transaction receipt information
- **Brands**: Brand management data

Database migrations are automatically applied on application startup.
